// carrier codes → AEB standard codes

A Streamlit app that reads a carrier's shipping document and maps its messy, house-specific event codes onto 31 standardized AEB event codes — using GPT for reading the document and a hybrid ML pipeline for the mapping itself.

What it does, in one screen

Every carrier labels events their own way. Eventmapper takes one row per carrier event and assigns the matching AEB code — attaching a confidence score and a source stamp that records which stage of the pipeline made the call, so every decision is traceable.

DEPOT 01

Get it running

# 1 · activate the venv (REQUIRED — system Python lacks the ML deps)
source venv/bin/activate        # Linux / Mac
.\venv\Scripts\activate         # Windows

# 2 · install dependencies
pip install -r requirements.txt

# 3 · provide your OpenAI key (.env in repo root)
echo "OPENAI_API_KEY=sk-..." > .env

# 4 · run the app
streamlit run app.py

# tests — 19 modules covering the pipeline (~25–30s)
pytest tests/ -v

First mapping run is slow: it embeds the 11k+ history rows once, then caches them to disk (examples/history_embeddings.npy). Later runs are fast.

Always use the venv

System Python is missing sentence-transformers and friends. Activate venv before any command, or imports fail.

SSL verification is off — on purpose

Disabled globally at startup for the corporate proxy. Don't "fix" it. The same workaround is mirrored in tests/conftest.py.

Cold-start a debug run

Sidebar → "Test: Mapping direkt" skips to Step 3 with a direct CSV/Excel upload — iterate on mapping without re-running extraction.

The 5-step pipeline

A sequential pipeline — each step produces the data the next one consumes, persisted in st.session_state and advanced by current_step. The backend mirrors it: one module per station, all under backend/.

0

Upload

backend/loader.py

Ingest PDF / Excel / CSV / TXT (and URL / JSON / XML) and extract the document text.

produces → raw_text
1

Analysis

backend/analyzer.py

LLM reads the document structure and identifies which tables hold status / reason codes.

produces → analysis_res (table candidates)
2

Extraction

backend/extractor.py

LLM pulls the data out of the selected tables into clean CSV strings.

produces → extraction_res (CSV)
3

Merge / Clean

backend/merger.py

Combine, de-duplicate and AI-transform the extracted rows into a tidy table.

produces → df_merged
4

Hybrid Mapping

backend/mapper.pythe core

Map every carrier row to one of the 31 AEB codes. This is where the real logic lives — detailed below.

produces → df_final (+ confidence + source)
DEPOT 04

Step 4 — why it's "hybrid"

No single technique maps logistics text reliably, so Step 4 stacks five — cheapest and most certain first, the LLM only as a last resort. A row exits the ladder the moment a stage is confident enough; the rest escalate. The source column records where each row got off.

1

History k-NN

Have we seen this exact text before? Cosine match against 11k+ cached embeddings, then weighted top-5 voting.

exit if sim ≥ 0.93
history-knn
2

Semantic + lexical pre-filter

Narrow 31 codes → top 10. Score = 0.7 · embeddings (text-embedding-3-large) + 0.3 · BM25.

→ top-10 candidates
3

Keyword boost

Each AEB code lists anchor keywords; every match lifts the score by up to +50%.

→ re-scored top-10
4

Cross-encoder re-ranking

A multilingual model (mmarco-mMiniLMv2) reads input and code together — far better at splitting close candidates.

exit if conf ≥ 0.60
emb+ce
5

LLM fallback

Only the genuinely-unsure rows. GPT decides with top-3 candidates, few-shot history, and a contrastive example — async, up to 15 concurrent.

final
llm-batch
The big idea

Spend cheap, certain methods first; reserve the expensive LLM for the handful of rows that truly need judgement — LLM accuracy where it matters, lookup speed everywhere else, every decision traceable via source + confidence. Knobs live in MAPPER_CONFIG (app.py) — thresholds, weights, feature flags, no hardcoded values in the matching logic.

HANDLING

House rules — read before you commit

Branch, never push to main

Create a feature branch for every change. main is protected.

Keep model IDs in sync

Changing an OpenAI model ID? Update all of: MODEL_CONFIG + PRICING in app.py, the model_name defaults in analyzer/extractor/merger, and the ID/price tests.

Touched EMB_DIMENSIONS?

Delete the disk cache (history_embeddings.npy, history_df.pkl, history_cache_meta.json) and clear the Streamlit cache so embeddings regenerate.

Don't touch the SSL workaround

It's deliberate, for the corporate proxy, and mirrored in tests. Removing it breaks model downloads and API calls.

DISPATCH

Your first change

Add a synonym

The smallest safe edit. Add a term to the right list in backend/synonyms.py — the analyzer picks it up in its prompts automatically.

backend/synonyms.py

Extend an AEB code

Add an entry to codes.py with an EN + DE description and a rich Keywords: line. Keywords meaningfully improve matching.

codes.py

Tune the mapper

Adjust thresholds and weights in MAPPER_CONFIG, or model constants at the top of backend/mapper.py. Validate against history before/after.
